## Premise

Hakan Ensari's fork (`hakanensari/OpenWhisperer`) branched from our exact HEAD and added 67
commits that **rewrite the app from the out-of-process Python stack to fully native Swift**:
WhisperKit (STT) + FluidAudio/Kokoro (TTS), in-process synth + gapless `AVAudioPlayerNode`
playback, in-process barge-in, and a **tagless voice-turn handshake** that replaces the `[VOICE:]`
tag. The Python venv, `setup.sh`, FastAPI server, and `mlx-audio` (whose version drift was a live
failure class) are deleted. Work is phased, gated (g2p-parity spike), and unit-tested.

**Verdict:** adopt it as the v1.5 base. It is the right architecture and directly serves the
complexity-reduction goal. Approved decisions (user, 2026-06-24):

1. **Adopt** by fast-forward merge of his branch (done — Phase 0).
2. **Distribution:** notarized direct download (Developer ID + Hardened Runtime), **not** the Mac
   App Store — the App Sandbox forbids this app's core mechanics (AX/CGEvent keystroke injection
   into other apps; writing hook scripts into `~/.claude`). No rewrite changes that.
3. **Tagless voice mode:** adopt (drops the `[VOICE:]` tag).
4. **UI:** full menubar reskin to match openwhisperer.com.

## Review adjudication (Opus, after architect-reviewer + swift-expert read-only review)

Both reviews confirmed the strategic calls (adopt rewrite, notarized direct download not MAS,
tagless) are correct and well-evidenced; no BLOCKERs. App Store non-viability **confirmed** (sandbox
forbids cross-app keystroke injection + `~/.claude` hook writes). Decisions:

**Acting on now (real, low-risk):**
- ✅ Deleted stale Python `__pycache__` in `scripts/` + `servers/` (removed empty `servers/`).
- ✅ Version bump 1.4.0→1.5.0 moved early (Info.plist ×2 + build-dmg.sh) — done.
- **F1** `TTSPlaybackController.play`: add `gen == generation` guard (not just `Task.isCancelled`)
  after `await synthesizeSamples`, and skip `synthFinished` when cancelled — immune to cooperative
  cancellation delays scheduling a stale chunk to a stopped engine.
- **F2** `TTSHTTPServer.receive`: 1 MB accumulated-body cap (OOM guard on the loopback server).
- **F3** `TTSHTTPServer`: set `NWListener.stateUpdateHandler` to surface a `.failed` bind (port in
  use) instead of silently showing "Running" with a dead endpoint.
- **F4** `AudioPlaybackEngine.schedule`: on `engine.start()` failure (e.g. device yanked mid-reply),
  notify the controller so it clears `tts_playing.lock` — avoids a stuck "Speaking…" state.
- **F5** `KokoroTTS.ensureVoicePack`: log download errors (was silently swallowed); `AVAudioFormat`
  force-unwrap → `guard … else fatalError` with message.
- **F7** Reconcile freshness windows: widen `voice-context.sh` `voice_turn` TTL 300s→900s to match
  `speak_pending` (fixes manual-submit "never speaks" after a pause); name the constants.

**Reskin (Phase 1) — incorporate swift-expert's exact guidance:**
- Add a semantic `OWColor` layer (`accent`, `surface`, `line`, `pillFill`, `pickerBg`,
  `pickerBorder`, `checkboxBorder`, `inkSoft`, `inkFaint`, `onAccent`, `success`) replacing the ~15
  `Color.primary.opacity(...)` / `Color(nsColor: .controlBackgroundColor)` / `.systemGray` sites
  across 8 structs (OWCard, OWInternalDivider, OWMenuPicker, OWCheckbox, ModernStatusRow,
  OWRowButtonStyle, OWPrimaryButtonStyle, root body).
- **Dark mode:** add a `Color(light:dark:)` helper via `NSColor(name:dynamicProvider:)`; every token
  gets a warm-dark variant (no light-only theme).
- **Surface:** replace `.ultraThinMaterial` with solid warm `surface`; override `NSWindow.backgroundColor`
  for seamless popover chrome in dark mode.
- **Contrast:** on-gold text uses ink `#2A2520`, not white (white on `#C0A06A` fails WCAG AA).
- **Fonts:** bundle **Fraunces** (OFL) for serif headers (`OWFont.title`/`sectionLabel`); keep body on
  the already-bundled Outfit / system; drop Hanken (imperceptible at menu sizes). Log
  `CTFontManagerRegisterFontsForURL` failures; register early.
- Keep semantic status colors (`.red` record, `.green` listening, `.orange` urgent); switch the TTS
  `.blue` playing indicator + "applied"/success `.green` button to gold `accent`. Replace fragile
  `stateColor == .green` equality with a state-derived bool.
- Discipline: **zero behavior change** — Phase 6 work-review must diff control wiring (no
  `action:`/`binding:` changes).

**Deferred / documented (not changing a working, tested merge without need):**
- `@MainActor` isolation of `ServerManager`/`DictationManager` + the `Thread.isMainThread` smell:
  current `dispatchPrecondition` + manual `MainActor.run` guards are correct; broad re-isolation is
  high-ripple. **Document as known-fragile; leave.**
- `:8000` request auth token, model-weight bundling in DMG, `prepareSTT` task-cancel-on-retry,
  `SubmitTrigger` rfind over-trim: **document as residual/fast-follow**, out of v1.5 scope.
- **Tagless limitation:** the hash matches only an **empty-buffer** dictation (pre-typed text or
  double-dictation diverges → treated as typed → silently unspoken); Codex gates on bare
  presence/freshness (weaker). **Document in README + AGENTS.md; add a non-empty-buffer note.** Inherent
  to the design, not a code bug.
- Confirmed already-present: `NSSpeechRecognitionUsageDescription` is in Info.plist (architect's
  hands-free concern moot). Cert-switch TCC re-grant + speech grant → added to "needs the user".
